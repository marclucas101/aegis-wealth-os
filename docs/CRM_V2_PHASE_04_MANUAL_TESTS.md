# CRM V2 Phase 04 — Manual Tests

1. [ ] Client sees own upcoming appointments only.
2. [ ] Client cannot access another client appointment URL.
3. [ ] Forged appointment ID returns not found without leakage.
4. [ ] Client can open request form and submit valid request.
5. [ ] Invalid request payload is rejected with safe validation message.
6. [ ] Repeated request with same idempotency key does not duplicate row.
7. [ ] Client can view proposed appointment and confirm it.
8. [ ] Client can decline proposal and request another time.
9. [ ] Client reschedule request preserves same appointment ID.
10. [ ] Client cancellation records client actor and retains row.
11. [ ] Adviser-only actions are unavailable to client.
12. [ ] Client can edit bounded topics; script tags are neutralized.
13. [ ] Client can complete only client/shared checklist items.
14. [ ] Adviser-only checklist items remain hidden.
15. [ ] Document upload entry links to secure existing vault route.
16. [ ] No storage paths or signed URLs shown in client appointment UI.
17. [ ] Published meeting summary appears only when explicitly published.
18. [ ] Unpublished outcomes remain hidden.
19. [ ] Notifications generated in-app only and contain safe metadata.
20. [ ] Mobile layout and keyboard navigation are usable.
